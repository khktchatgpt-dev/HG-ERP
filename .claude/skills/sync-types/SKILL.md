# sync-types

Đồng bộ TypeScript types từ Supabase Postgres schema vào project. Sau khi chạy, mọi `db().from('table')` sẽ được suy ra type tự động.

## Khi nào chạy

- Vừa apply migration mới (tạo/đổi/xoá table, column, enum)
- Repo nhận `any` từ `.from('...')` thay vì type đúng
- User yêu cầu rõ ràng ("sync types", "regen types")

## Quy trình

1. **Kiểm tra schema hiện tại** — gọi `mcp__supabase__list_tables` với `schemas=["public"]` để xác nhận có table và biết structure cần expect.

2. **Generate types** — gọi `mcp__supabase__generate_typescript_types`. Tool trả về một chuỗi TS đầy đủ `export type Database = { public: { Tables: { ... }, Views: { ... }, ... } }`.

3. **Ghi file** — đặt nguyên chuỗi vào `src/lib/database.types.ts`. KHÔNG sửa tay file này — nó là output máy. Thêm comment header:
   ```ts
   // AUTO-GENERATED by .claude/skills/sync-types — do not edit by hand.
   // Re-run: ask Claude to "sync types" after applying a migration.
   ```

4. **Verify wiring** — đọc `src/server/db.ts`. Phải có:
   ```ts
   import type { Database } from '@/lib/database.types'
   ...
   return createClient<Database>(url, key, { ... })
   ```
   Nếu thiếu generic, sửa lại. Nếu đã có rồi, bỏ qua.

5. **Typecheck** — chạy `npm run typecheck`. Nếu repo nào hỏng type vì schema đổi (column bị xoá/đổi tên), liệt kê file:line cho user — KHÔNG tự sửa repo, vì có thể là intentional rename phía DB hoặc bug cần user quyết.

6. **Báo lại** — 1-2 câu: "Đã sync, N tables, M views. Typecheck pass/fail." Không liệt kê toàn bộ schema.

## Lưu ý

- File `database.types.ts` thường rất lớn → ghi bằng Write, không Edit.
- Nếu MCP supabase chưa connect (`mcp__supabase__*` không có trong tool list), báo user: "Cần connect MCP Supabase trước — xem CLAUDE.md section MCP."
- Không commit `database.types.ts` vào cùng PR với migration nếu user chưa apply migration lên remote — types sẽ stale so với DB thật.
